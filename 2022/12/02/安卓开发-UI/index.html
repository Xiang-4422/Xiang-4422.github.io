布局

布局是一种可以用来放置很多控件的容器，它可以按照一定的规律调整内部控件的位置，从而编写精美的界面。同时布局也可以嵌套布局。

布局和控件之间的关系：

  • 线性布局（LinearLayout）

    线性布局中的控件线性排列。

    • 通过android:orientation="vertical/horizontal"来控制水平垂直/水平排列。
    • 通过android:layout_weight来控制控件所占屏幕比例

  • 相对布局（RelativeLayout）

    相对布局中可以设置控件之间的相对位置关系。

  • 帧布局（FrameLayout）

    新加入的控件永远在前一个控件之上，都放在左上角。

  • 绝对布局（AbsoluteLayout）

    通过坐标来控制控件的位置，左上角为（0，0），已废弃。

  • 表格布局（TableLayout）

    将子元素拍成行和列，由多个TableRow对象组成，每个TableRow代表一行。

    TableLayout的子节点不能设置layout_width属性，宽度总是MATCH_PARENT。

  • -约束布局（ConstraintLayout）

    每个控件有水平和垂直两个方向上的约束，共四个约束。（有点像机械制图中的约束）

    ConstraintLayout中的控件不建议使用MATCH_PARENT，建议使用MATCH_CONSTRAINT（0dp）和水平约束来设置。

  • FlexboxLayout

    对应于CSS中的CSS Flexible Box Layout Module。

ListView

ListView的使用

  1. 在Activit的布局中引入ListView，并为其指定一个id

  2. 编写自定义ListView子项

  3. 编写Adapter

    • 先定义一个实体类用于包装ListView子项中需要的数据

    • 创建一个自定义适配器，继承自ArrayAdapter，并将泛型指定为Fruit类

      这里的inflate方法有三个参数：

      1. ListView子项布局的id
      2. 指定的父布局
      3. false表示：让我们在父布局中声明的layout属性有效，但不会为这个View添加父布局。因为一旦View有了父布局后，它就不能添加到ListView中了。

  4. 在Activity中为listView注入Adapter

ListView的优化

在FruitAdapter的getView()方法中，每次都将布局重新加载一遍，当ListView快速滚动时，就会成为性能瓶颈。

优化有两个方面：

  1. 通过getView方法中的convertView参数来缓存View
  2. 使用ViewHolder来缓存View中的控件实例

RecyclerView

listView有两个方面的缺点：

  • 需要使用一些技巧来提升其运行效率，否则其性能很差
  • 扩展性不足，例如不支持横向滚动

RecyclerView可以说是一个增强的ListView，不经可以实现和ListView相同的效果，还优化ListView存在的各种不足之处。

RecyclerView的基本用法

  1. 添加RecycleView的依赖

    在app/build.gredle文件中，在dependencies闭包中添加依赖。

  2. 在布局中添加RecycleView的控件，为其指定id。

  3. 跟使用ListView一样，新建Fruit类和fruit_item.xml子布局。

  4. 为RecycleView准备一个适配器类Adapter，其中ViewHolder是适配器中定义的一个内部类

  5. 在Activity中为RecycleView注入LayoutManager和Adapter

横向滚动和瀑布流布局

RecyclerView的布局由设置的LayoutManager控制。

横向滚动

  1. 在子项布局中，将子项内部元素的排列方式改为垂直

    修改子项布局主要是为了适配横向滚动

  2. 在Activity中对LayoutManager进行设置，设置为水平

瀑布流布局

除了上面使用的LinearLayoutManger之外，RecyclerView还提供了GridLayoutManager（网格布局）、StaggeredGridLayoutManager（瀑布流布局）。

  1. 修改子项布局

    layout_margin：改为match_parent，因为瀑布流布局宽度由布局的列数决定自动适配

    layout_margin：为每项之间留点间距

  2. 在Activity中使用StaggeredGridLayoutManager来控制RecyclerView的布局

    参数一：指定列数

    参数二：指定布局排列方向

RecyclerView的点击事件

不同于ListView，RecyclerView并没有提供类似于setOnItemClickListener()这样的注册监听器方法，而是需要我们自己给子项的View注册点击事件。

RecyclerView这样设计的原因：

  • ListView中的setOnItemClickListener()注册的是子项的点击事件，如果我们想点击子项中具体的某个按钮，那么ListView实现起来就很麻烦。
  • 因此RecyclerView就直接抛弃了子项点击事件的监听器，所有的点击事件由具体的View去注册。

修改Adapter，在Adapter的onCreatViewHolder中通过ViewHolder为子项中的元素注册监听器

自定义控件

View继承结构

控件和布局的继承结构：

由上可见，所有的控件和布局都是直接或间接地继承自View的。View是Android中最基本的一个UI组件，它可以在屏幕上绘制一块矩形区域，并响应这块区域中的各事件。

  • 我们使用的各种控件其实就是在View的基础之上添加了各自特有的功能。
  • 而ViewGroup则是一种特殊的View，它是一个用于放置控件和布局的容器，它可以包含多个子View和子ViewGroup。

自定义控件的方法

自定控件主要有两种方法：

  • 方法一：先编写XML的自定义布局，在其他界面中引用该XML布局
  • 方法二：创建一个控件类，在类中动态加载自定义的布局，并且同时还可以为自定义布局绑定逻辑，然后在其他界面引用这个控件类。
