安卓开发-四大组件-Activity

本文最后更新于：2022年12月2日 晚上

活动简介

活动（Activity）是一种可以包含用户界面的组件，主要用于与用户进行交互。一个应用程序可以包含零个或多个活动，不包含活动就是一个没有用户界面的应用。

活动创建步骤

1、编写布局（first_layout.xml）

Android 程序的设计讲究逻辑与视图分离，最好每一个活动都能对应一个布局。布局是用来显示界面内容的。

在 app/src/main/res/layout 中 右键->New->Layout resource file 即可创建一个布局。

具体布局的编写会在后面的 UI 介绍中详细说明。

2、创建活动（FirstActivity.java）

在 java 文件夹下的应用程序包中 New->Activity->Empty Activity 即可新建一个空的活动。

在新建的时候会有两个可选的选项：

  • Generate Layout File：自动生成对应的 layout 布局文件
  • Launcher Activity：将新建的活动设置为主活动

新建后的Activity文件：

3、注册活动（AndroidManifest.xml）

在新建Activity的时候，Android Studio 会自动在 AndroidManifest文件中注册我们新建的活动。

  • 在<application>标签中使用<activity>标签注册活动
  • <activity>标签中使用 android:name 属性来指定具体注册的活动
  • 在 <activity>标签中使用<intent-filter>标签指定action和category可以指定该活动为主活动

Toast

Toast是Android系统提供的一种非常好的提醒方式，在程序中可以使用它将一些短小的消息通知给用户，这些消息会在一段时间后显示，并且不会占用任何屏幕空间。

通过Toast.makeText().show();来显示一条Toast通知。

makeText()方法需要三个参数：

  1. Context（上下文对象）

    活动本身就是一个Context对象，所以在活动中调用Toast可以直接传入this

  2. Charsequence（文本内容）

  3. int（显示时长）

    可以使用Toast自带的常量：

    • Toast.LENGTH_SHORT
    • Toast.LENGTH_LONG

Intent

通过Intent可以在活动之间跳转。

Intent大致可以分为两种：显示Intent和隐式Intent。

显示Intent

使用Intent（Context packageContext, Class<?> cls）即可创建一个显示Intent。

  • 参数一：Context要求提供一个启动活动的上下文
  • 参数二：Class指定想要启动的目标活动

隐式Intent

隐式Intent并不明确指定想要启动哪一个活动，而是指定一系列action和category等信息，然后由系统分析该Intent，匹配合适的活动启动。

指定Activity响应的action和category

Activity在AndroidManifest.xml中注册的时候，可以指定当前活动可以响应的action和category。

启动隐式的Intent

使用intent(String action)来创建一个隐式Intent，然后可以使用addCategoty(String category)指定category。

  • 隐式Intent只能指定一个action，但是可以指定多个category
  • “android.intent.category.DEFAULT”是intent创建时的默认category
  • 只有action和category与intent全部匹配的activity才会被启动

使用隐式Intent启动其他程序活动

例如启动系统浏览器：

  • Intent.ACTION_VIEW 是 Android系统内置的动作，其常量值为android.intent.action.VIEW
  • 在通过Uri.parse()方法将网站地址解析为一个Uri对象
  • 在调用Intent的setData()方法将Uri对象设置到intent中
  • 最后启动，即可打开系统浏览器

让自己的Activity响应其他程序发出的隐式Intent

在Activity注册的时候，在<intent-filter>标签中再配置一个<data>标签，用于更精准地指定当前活动能够响应什么类型地数据。

<data>标签中可以配置的属性：

  • android:scheme。指定数据地协议。如http、geo、tel。
  • android:host。指定数据的主机名。如www.google.com。
  • android:port。指定数据的端口，一般紧随在主机名后。
  • android:path。指定主机名和端口后的部分，如一段网址中跟在域名后的部分。
  • android:mimeType：指定可以处理的数据类型。

活动的生命周期

返回栈

Android使用任务（Task）来管理活动，一个任务就是一组存放在栈里的活动的集合，这个栈也被称为返回栈（Back Stack）。

栈是一种后进先出的数据结构，当启动一个新的活动时，它被压入栈中位于栈顶，当我们按下Back键或调用finish方法去销毁一个活动，处于栈顶的活动会出栈，这时前一个入栈的活动就会位于栈顶。系统总是会显示处于栈顶的活动给用户。

图示：

活动的状态

  1. 运行状态

    当活动位于返回栈的栈顶时，该活动处于运行状态。

  2. 暂停状态

    当活动不处于栈顶，但仍然可见时，该活动处于暂停状态。

  3. 停止状态

    当活动不处于栈顶，并完全不可见时，该活动处于停止状态。

  4. 销毁状态

    当活动从返回栈中移除后，该活动处于销毁状态。

系统会根据活动状态选择优先回收的活动。

活动的生存期

Activity类中定义的7个回调方法，一一对应活动生命周期的每一个环节：

  1. onCreate()：活动第一次被创建时调用。

  2. onStart()：活动由不可见变为可见时调用。

  3. onResume()：方法准备好和用户进行交互时调用。

    此时活动一定处于栈顶，并处于运行状态

  4. onPause()：当系统要启动或恢复另一个活动时调用。

    一般在此方法中释放资源，保存关键数据，不要在该方法中进行耗时操作，会影响新栈顶活动的调用

  5. onStop()：当活动完全不可见时调用。

    它和onPause()方法的主要区别在于，若启动的新活动是一个对话框式的活动，那么onPause()会被执行。

    （在Activity注册时，为Activity设置属性android:theme="@style/Theme.AppCompat.Dialog"即可将Activity设置为对话框式的活动）

  6. onRestart()：在方法由停止状态变为运行状态之前调用。

  7. onDestoty()：在活动被销毁之前调用。

根据活动的生命周期，又可以将活动分为三种生存期：

  • 完整生存期：onCreat()和onDestory()方法之间所经历的。

    一般来说，在onCreate()中完成各种初始化操作，而在onDestroy()方法中完成释放内存的操作。

  • 可见生存期：onStart()和onStop()方法之间所经历的。

    在可见生存期间内，互动对用户总是可见的，即使可能无法与用户进行交互。我们可以通过这两个方法管理对用户可见的资源。

    例如，在onStart()方法中对资源进行加载，在onStop()方法中对资源进行释放，从而保证处于停止状态的活动不会占用过多内存。

  • 前台生存期： onResum()和onPause()之间经历的。

    在前台生存期，活动总是处于运行状态，并可以和用户进行交互，我们平时看到和接触到最多的就是这个状态下的活动。

活动的启动模式

活动共有四种启动模式，通过给<activity>标签指定android:launchMode 属性来选择启动模式。

standard（默认）

每次启动一个Standart活动，都会创建一个该活动的一个新的实例压入返回栈中。

singleTop

在启动一个singleTop活动的时候，若该活动已经处于栈顶，则会直接使用它，不会再创建一个新的活动实例压入栈顶。若栈顶不是该活动，则会新建一个活动入栈。

singleTask

在启动一个singleTask活动的时候，若栈中已经存在该活动，则会将该活动上的所有其他活动全部出栈，使该活动处于栈顶。若栈中没有改活动，则会新建一个活动入栈。

singleInstance

在启动一个singleInstance活动时，会启用一个新的返回栈来管理这个活动。

这样做的意义：若我们程序中的一个活动允许其他程序调用，使用singleInstance模式启动活动可以共享这个活动的实例。因为每个程序都有自己的返回栈，若使用其他三种启动模式，同一个活动在不同的返回栈中入栈必然是创建了一个新的实例，就无法是实现实例共享。而singleInstance模式下会为该活动创建一个单独的返回栈，这样不管哪个应用访问该活动那个，都公用的是用一个返回栈，也就解决了活动实例的共享问题。

活动使用技巧

根据界面判断活动

自己写的代码可能清楚界面对应的活动，而在企业中可能接手的是别人写的代码，当需要在某界面上修改一些对应Activity代码是，可能半天找不到对应的活动是哪一个。

写一个BaseActivity，并让所有的Activity都继承自BaseActivity。

这样当进入到一个界面的时候，该活动的类名就会被打印出来。
