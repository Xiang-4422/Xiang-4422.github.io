安卓开发-总体简介

本文最后更新于：2022年12月2日 下午

安卓系统架构

1、Linux 内核层

安卓平台的基础是Linux内核。

  • ART依靠linux内核执行底层功能。例如线程和内存管理。
  • 硬件制造商在该层为硬件开发底层驱动。

2、硬件抽象层

硬件抽象层（HAL）提供标准界面，向更高级别的Java API框架显式设备硬件功能。

HAL包含多个库模块，其中每个模块都为特定硬件组件实现一个界面，例如相机、蓝牙模块。当框架API要求访问硬件设备时，系统将为该硬件组件加载库模块。

3、系统运行时库层

该层包含很多C/C++编写的原生库还有Android Runtime。

ART

每个应用都在其自己的进程中运行，都单独拥有一个ART实例。

ART编写为通过执行DEX文件来运行多个虚拟机。DEX文件是专为Android设计的字节码格式，经过优化，使用内存很少。编译工具链（如Jack）可以将Java源码编译为DEX字节码，使其在Android平台运行。

ART的主要功能：

  • 预先编译（AOT）和即时编译（JIT）
  • 垃圾回收（GC）
  • DEX字节码转化为机器码
  • 更好的调试支持

安卓还包含一套核心运行时库，可提供给 Java API 框架 Java 编程语言的大部分功能。

原生C/C++库

许多核心Android系统组件和服务（例如ART、HAL）构建自原生代码。Android 平台提供 Java API框架以向应用层提供部分原生库的功能。

例如，可以通过 Android 框架的 Java OpenGL API 访问 OpenGL ES， 以支持在应用中绘制和操作 2D 和 3D 图形。如果开发的是需要 C/C++ 代码的应用，可以使用 Android NDK 直接从原生代码访问某些原生平台库。

4、API框架层

应用可以调用通过 Java 语言编写的 API 从而使用 Android OS的整个功能集。

这些 API 形成创建 Android 应用所需的构建块，它们可以简化核心模块化系统组件和服务重复使用。

包含的组件和服务：

  • 视图系统

    用于构件应用的UI，包括列表、网格、文本框、按钮以及可嵌入的网络浏览器。

  • 资源管理器

    用于访问非代码资源，例如本地化的字符串、图形和布局文件。

  • 通知管理器

    可让所有应用在状态栏中显式自定义提醒。

  • 活动管理器

    用于管理应用的生命周期，提供常见的导航返回栈。

  • 内容提供程序

    可让应用访问其他应用中的数据，或共享本应用的数据。

5、应用层

Android系统自带很多核心应用，例如邮件、短信、日历、浏览器、联系人等。

系统自带应用和第三方应用一样，没有特殊状态。因此可使用一些第三方应用替换系统自带应用，系统应用可作为用户的应用，以及提供开发者访问，例如若第三方软件需要发送短信，可以直接调用系统的短信应用，而无需自己构建该功能。

安卓应用开发的特色

四大组件

  • 活动（Activity）

    活动是安卓应用的门面，在应用中看得见的都放在活动中。

  • 服务（Service）

    服务在后台运行，无法看到，应用退出后仍可以运行。

  • 广播接收器（Broadcast Receiver）

    广播接收器允许应用接收来自各处的广播，例如电话、短信等。同时应用也可以对外发送广播消息。

  • 内容提供器（Content Provider）

    内容提供器为跨应用分享数据提供可能，例如读取系统电话簿应用中的联系人信息。

系统控件

系统提供了丰富的系统控件用于编写界面。同时也可以定制自己的控件使用。

SQLite数据库

安卓系统自带了SQLite数据库。

其特点：

  • 轻量化、速度快、嵌入式、关系型数据库
  • 支持标准SQL语法
  • Android封装了API用于操作该数据库

项目结构

基本无需修改文件

  • .gradle 和 .idea文件夹

    由 Android Studio自动生成，无需关心，无需编辑。

  • gradle文件夹

    包含了 gradle wrapper 的配置文件。Android会根据本地的缓存情况自动决定是否下载 gradle wrapper。默认没有启用 gralde wrapper。

  • .gitignore

    用于将指定目录或文件排除在版本控制之外。

  • build.gradle

    项目全局的gredle构建脚本。通常不需要修改。

  • gradle.properties

    全局的gradle配置文件，这里配置的属性会影响到项目中所有的gradle编译脚本。

  • gradlew和gradlew.bat

    用于在命令行界面执行gradle命令。

  • local.properties

    用于指定本机中 Android SDK 路径，通常自动生成，不需要修改。

  • settings.gradle

    用于指定项目中所有引用的模块。通常情况下模块应用都是自动完成的，需要手动修改的场景较少。

app文件夹

基本无需修改文件

  • build文件夹

    类似于外层builde目录，包含一些编译时自动生成的文件。

  • libs文件夹

    项目中使用到的第三方jar包放在该目录下。在该目录中的jar包会被自动添加到构建路径中。

  • .gitignore

    同外层.gitignore文件一样，该文件用于将app模块中指定的目录或文件排除在版本控制之外。

  • proguard-rules.pro

    用于指定项目代码混淆规则，防止代码被破解。

src文件夹

androidTest 和test 文件夹

这两个目录用于存放编写的测试用例，一个是Android test测试用例，一个是Unit test测试用例。

main 文件夹

该文件夹中存放的就是我们真正编写的安卓应用代码。

1. java

此处就是用于存放我们所有的Java代码。

2. res

此处用于存放项目中使用到的资源。

包含的文件夹内容：

  • layout：布局文件
  • values：字符串、样式、主题等配置
  • drawable：图片
  • mipmap：应用图标
  • dpi后缀文件夹：设备适配（一般使用xxhdpi）

引用res中资源的方式：

  • 在Java代码中引用：通过R.string.app_name可获取上面字符串的引用。
  • 在XML文件中引用：通过@string/app_name可获取上面字符串的引用。

其中的string可以替换为drawable、mipmap、layout用于引用图片、图标、布局。

3. AndroidManifest.xml

Android项目的配置文件。

  • 在项目中定义的四大组件需要在该文件中注册
  • 应用的所需权限在该文件中声明

build.gradle

日志工具

安卓提供的日志工具时Log（android.util.Log）。

Log提供了五种方法用于打印日志，分别对应不同的日志级别：

  • Log.v()

    verbose：用于打印最为琐碎、意义最小的日志信息。

  • Log.d()

    debug：用于打印一些对调试程序和分析问题有用的调试信息。

  • Log.i()

    important：用于打印一些比较重要的数据。这些数据应该是你想要看到的、可以帮助你分析用户行为的数据。

  • Log.w()

    用于打印一些警告信息。提示程序潜在的风险，最好去修复出现警告的地方。

  • Log.e()

    用于打印程序中的错误信息。当出现错误信息时（例如进入catch块），一般代表程序出现严重问题，必须尽快修复。

方法所需的参数：

  • String tag：一般传入当前类名即可，主要用于对打印信息进行过滤。
  • String msg：传入想要打印的具体内容。

Log的快捷使用：

  • 在java类中输入logt，再按下Tab键，就会以当前类名生成一个TAG常量。
  • 输入logd，再按下Tab键，会自动补全一条log语句，tag参数默认是TAG常量。

为什么不使用 System.out.println 打印日志?

因为除了方便之外，全是缺点：

  • 日志打印不可控
  • 打印时间无法确定
  • 不能添加过滤器
  • 日志没有级别区分
