MySQL入门笔记

本文最后更新于：2022年9月2日 晚上

SQL简介

  1. 什么是 SQL

    结构化查询语言（Structured Query Language）

  2. SQL 的作用

    客户端使用SQL来操作服务器

  3. 语法规范

    • SQL语句可以在单行或多行书写, 以分号结尾
    • 可以使用空格和缩进来增强语句的可读性
    • MySQL不区分大小写, 但是建议使用大写

  4. SQL语句分类

    • DDL(Data Definition Language): 数据定义语言

      创建、删除、修改：库、表结构

    • DML(Data Manipulation Language): 数据操作语言

      增、删、改：表记录

    • DQL(Data Query Language): 数据查询语言

      用来查询记录（数据）

    • DCL(Data Control Language): 数据控制语言

      用来定义访问权限和安全级别（用户创建，以及用户的权限管理）

DDL（数据定义语言）

操作数据库

  • 查看所有数据库：SHOW DATABASES
  • 切换（选择要操作的）数据库：USE 数据库名
  • 创建数据库：CREATE DATABASE [IF NOT EXISTS] 数据库名
  • 删除数据库：DROP DATABASE [IF EXISTS] 数据库名
  • 修改数据库编码：ALTER DATABASE 数据库名CHARACTER SET 编码名

数据类型（列数据类型）

  • int：整型

  • double：浮点型

    double(5, 2)表示最多5位，其中必须有两位小数

  • decimal：浮点型

    十进制浮点类型，不会出现浮点数精度丢失问题，在跟钱相关的方面使用该类

  • char：固定长度字符串

    char(255)，数据长度不足，则会补足到指定长度

  • varchar：可变长字符串类型

    • varchar(65535)，最大指定长度为65535，当数据长度不足指定长度的时候，不用补足到指定长度。

    • 需要使用额外的一个字节来记录数据长度

  • text：字符串类型

    MySQL的方言，在标准SQL中为clob

    • tinytext：256B
    • text：64K
    • mediumtext：16M
    • longtext：4G

  • blob：字节类型

    • varblob
    • tinyblob
    • blob
    • mediumblob
    • longblob

  • 时间相关

    • date：日期类型，格式：yyyy- MM-dd;
    • time：时间类型， 格式：hh:mm:ss；
    • timestamp：时间戳类型：

  2. 查看表

    • 查看当前数据库中左右表名称：SHOW TABLES;

    • 查看指定表的创建语句：SHOW CREATE TABLE 表名;(了解)

DCL（数据控制语言）

对于DCL的理解：

  • 一个项目创建一个用户，一个项目对应的数据库只有一个
  • 这个用户只拥有该数据库的权限，无法操作其他数据库

1. 创建用户

用户只能在指定IP登录

用户可以在任意IP登录

2. 给用户授权

  • 将指定数据库的指定权限赋予指定用户

  • 要素：权限、数据库、用户

    权限：

    • CREATE、ALTER、DROP、INSERT、UPDATE、DELETE、SELECT

    • ALL表示所有权限

3. 撤销授权

DQL（数据查询语言）

一、基础查询

  • 查询所有列（整张表）

1. 列控制

  1. 查询指定列
  1. 完全重复的记录只有一次

当查询结果中多行记录一摸一样时，只显示一行

  1. 列运算

  • 数量类型的列可以做加减乘除运算

  • 字符串类型的列可以做连续运算

  • 转换NULL值

IFNULL(列名1，0) 表示，如果列1中的值为NULL，则当为0计算

  • 给列起别名

在使用列运算后自动生成的列名称不直观，可以给列名起个别名(AS可省略)

2. 行控制

  1. 条件查询: WHERE

    同前面的UPDATE和DELETE语句一样，使用WHERE子句来控制记录

  2. 模糊查询: LIKE

    用来对字符串列进行模糊查询

    • _下划线，用来匹配一个字符
    • %百分号，用来匹配0-n个字符

二、排序：ORDER BY

  1. 默认升序: ASC

  2. 降序排序: DESC

  3. 多条件排序

三、聚合函数

  1. 查询有效行数：COUNT()

    括号中可以是列名、*、数字，其中*和数字跟*等效

  2. 计算

    计算中的括号中只能填写数字列

    • 总和：SUM()
    • 最大值：MAX()
    • 平均值：AVG()

  3. 综合计算

四、分组查询：GROUP BY|HAVING

  • 分组

将一列中值相同的行分为一组

  • 查询

分组查询不能使用个人信息查询，都是使用聚合信息进行查询

分组查询过滤

  • 分组前过滤条件（对整张表进行过滤）

    WHERE

    在ORDER BY之前

  • 分组后过滤条件（对分组查询的结果过滤）

    HAVING

    在ORDER BY之后

同一SQL语句中，关键字使用顺序(执行顺序也是如此)：

  1. SELECT
  2. FROM
  3. WHERE
  4. GROUP BY
  5. HAVING
  6. ORDER BY
