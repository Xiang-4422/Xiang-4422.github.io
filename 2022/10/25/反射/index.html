Class类

处理int等基本类型外，Java的其他类型全部都是class（包括interface）。

例如：

  • String
  • Object
  • Runable
  • Exception
  • …

所以说class的本质就是数据类型（Type）。无继承关系的数据类型无法赋值。

反射

由于JVM为每个加载的class创建了对应的Class实例，并在实例中保存了该class的所有信息，包括类名、包名、父类、实现的接口、所有方法、字段等。因此，如果获取了某个Class实例，我们就可以通过这个Class实例获取到该实例对应的class的所有信息。

这种通过Class实例获取class信息的方法称为反射（Reflection）。

获取Class实例的方法：

  • 通过class的静态变量class获取

  • 通过实例变量的getClass()方法获取

  • 根据class的完成类名，通过静态方法Class.forName()获取

因为Class实例在JVM中是唯一的，所以不同方法获取的Class实例是同一个。

Class实例和instanceof的区别：

  • instanceof不但匹配指定的类型，还匹配指定类型的子类
  • 使用==判段class实例可以精确地判断数据类型，不能做子类型比较

访问字段

获取字段信息

通过Class实例获取字段信息的方法：

  • Field getField(name)：根据字段名获取当前类某个public的field（包括父类）
  • Field getDeclaredField(name)：根据字段名获取当前类的某个field（不包括父类）
  • Field[] getField()：获取所有的public的field（包括父类）
  • Field[] get DeclaredFields()：获取所有的field（不包括父类）

Field对象包含一个字段的所有信息：

  • getName()：返回字段名称。返回值为String
  • getType()：返回字段类型。返回值也是一个Class实例
  • int getModifiers()：返回字段的修饰符。返回值是一个int，不同bit表示不同的含义

反射破坏封装：

通过反射，可以获取对象的private字段的值，这破坏了类的封装。

既然反射和可以破坏封装，那么封装的意义在哪儿？

  • 正常情况下，我们总是通过p.name来访问Person的name字段，编译器会根据字段的访问权限来决定是否允许访问字段，这就达到了数据封装的目的。
  • 而反射是一种非常规的用法，使用反射获取字段值，首先代码繁琐，其次，它更多地是给工具或者底层框架来使用的，目的是在不知道目标实例对象任何信息的情况下，获取特定字段的值。
  • setAccessible(true)不是一定成功。如果JVM运行期存在SecurityManager，那么它会根据队则进行检查，可能会阻止对某些类（例如以java和javax开头的package中的类）调用setAccessible(true)，这样可以保证JVM核心库的安全
